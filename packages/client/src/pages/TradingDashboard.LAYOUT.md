## Component Dimensions

### Desktop
- **Total Width**: 100vw - padding
- **Chart Area**: 66.67% (8/12 columns)
- **Sidebar**: 33.33% (4/12 columns)
- **Gap**: 16px between all components
- **Card Height**: Auto (content-based)
- **Chart Height**: 500px
- **OrderBook Height**: 400px
- **Positions Height**: Auto (max 300px with scroll)
- **OrderForm Height**: Auto

### Tablet
- **Total Width**: 100vw - padding
- **Stats Cards**: 2 columns
- **All Components**: Full width (1 column)
- **Gap**: 16px

### Mobile
- **Total Width**: 100vw - padding
- **All Components**: Full width (1 column)
- **Gap**: 16px
- **Chart Height**: 300px (reduced)
- **OrderBook Height**: 300px (reduced)

## Interaction Points

1. **Symbol Selector** (Chart)
   - Dropdown with 5 options
   - Syncs with OrderBook and OrderForm

2. **WebSocket Indicator** (Header)
   - Visual-only, no interaction
   - Updates automatically

3. **Stats Cards** (Top)
   - Display-only, no interaction
   - Update in real-time

4. **TradingChart** (Main)
   - Zoom/pan functionality (from lightweight-charts)
   - Timeframe selector buttons

5. **OrderBook** (Sidebar)
   - Scrollable list
   - Click price to fill OrderForm (future)

6. **PositionsList** (Bottom Left)
   - Close position button per row
   - Sortable columns (future)

7. **OrderForm** (Bottom Right)
   - All form inputs interactive
   - Validation feedback
   - Confirmation dialog

## Performance Considerations

### Optimization Techniques Used
1. **useMemo** for expensive calculations (balance, P&L)
2. **Component memoization** (future enhancement)
3. **Throttled updates** in OrderBook (500ms)
4. **Debounced inputs** in OrderForm
5. **Virtualized lists** for large position lists (future)

### Bundle Size
- TradingDashboard: ~5KB (gzipped)
- All Epic 5 components: ~45KB (gzipped)
- lightweight-charts library: ~80KB (gzipped)
